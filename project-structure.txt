# Iglesia Betel - Project Documentation

## Philosophy & Architecture
This project is a static website built with Hugo. It is designed to be fast, simple to maintain, and editable by non-technical users through Decap CMS.

### Key Concepts
1. Static site generation with Hugo.
2. Global configuration in `hugo.toml`.
3. Editable content in `content/`.
4. Shared templates in `layouts/`.
5. Decap CMS integration in `static/admin/`.

## Configuration
- `hugo.toml`: Core Hugo settings plus `[params]` for contact details, schedules, and other site-wide values.

## Content Structure

### `/content/predicaciones/`
Contains one Markdown file per sermon.

Typical front matter fields:
- `title`
- `date`
- `predicador`
- `serie`
- `serie_ep`
- `pasaje`
- `duracion`
- `youtube_url`
- `youtube_id`
- `destacado`
- `tags`

### `/content/ministerios/`
Contains one Markdown file per ministry.

Typical front matter fields:
- `title`
- `weight`
- `icon`
- `color`
- `dia`
- `hora`
- `edad`
- `destacado`

### `/content/nosotros/_index.md`
Stores the editable content for the Nosotros section:
- `description`
- `vision`
- `mision`
- `historia`
- `afiliacion`

## JavaScript Functionality

`static/js/main.js` handles:
- Mobile navigation toggle
- FAQ accordion behavior
- Client-side filtering for predicaciones
- Netlify Identity redirect to `/admin/`

## Main Templates

- `layouts/index.html`: Home page
- `layouts/nosotros/list.html`: Nosotros section list view
- `layouts/nosotros/single.html`: Nosotros single template
- `layouts/predicaciones/list.html`: Sermons archive
- `layouts/predicaciones/single.html`: Sermon detail
- `layouts/contacto/single.html`: Contact page
- `layouts/partials/nav.html`: Main navigation
- `layouts/partials/footer.html`: Footer

## Notes

- `anios_ministerio` is defined in `hugo.toml` and used in the home page and Nosotros templates.
- The `public/` directory contains generated output and can be rebuilt from Hugo.
